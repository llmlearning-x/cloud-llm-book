# 4.1 RAG 核心原理与知识库构建

## 场景引入

某大型制造企业拥有数万份设备维护手册、故障案例和维修记录。技术人员在排查故障时，往往需要花费数小时翻阅文档。通过构建基于 RAG（检索增强生成）的企业知识库，可以将查询时间缩短至秒级，并显著提升回答的准确性。

## 4.1.1 RAG 工作流程

RAG 系统由“离线构建”和“在线检索”两个核心阶段组成：

1.  **文档处理**：解析 PDF/Word → 文本清洗 → 语义分块（Chunking）。
2.  **向量化**：使用 Embedding 模型将文本块转换为高维向量。
3.  **存储**：存入向量数据库（如阿里云 DashVector）。
4.  **检索增强**：用户提问 → 问题向量化 → 相似度检索 → 拼接 Prompt → 大模型生成。

## 4.1.2 文档切分策略

合理的切分是提升检索精度的关键。常见的策略包括：

*   **固定长度切分**：简单但容易切断语义。
*   **递归字符切分**：按段落、句子、单词的顺序尝试切分，保持语义完整性。
*   **结构化切分**：利用 Markdown 标题或 PDF 目录结构进行逻辑分割。

```python
from langchain.text_splitter import RecursiveCharacterTextSplitter

splitter = RecursiveCharacterTextSplitter(
    separators=["\n\n", "\n", "。", "！"],
    chunk_size=300,
    chunk_overlap=50
)
chunks = splitter.split_documents(cleaned_docs)
```

## 4.1.3 阿里云百炼知识库实战

阿里云百炼平台提供了一站式的知识库管理功能：

*   **多格式支持**：自动解析 Word、PDF、Excel 等常见办公文档。
*   **智能分片**：内置针对中文优化的分片算法。
*   **混合检索**：同时支持关键词匹配和向量语义检索，大幅提升召回率。

> **技术决策点**：对于非结构化数据占比高的企业，建议优先采用百炼平台的托管知识库服务，以减少底层向量库的运维投入。
