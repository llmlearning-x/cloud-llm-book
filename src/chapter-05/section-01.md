# 5.1 Agent 核心概念与规划算法

## 场景引入

某电商企业希望实现一个“智能运营助理”，能够根据指令自动完成“查询上周销量、生成图表并发送邮件”的复杂任务。这不再是一个简单的问答系统，而是一个具备自主规划和工具调用能力的 AI Agent（智能体）。

## 5.1.1 Agent 的四层架构

*   **感知层**：接收文本、图像等多模态输入。
*   **记忆层**：维护短期对话历史和长期用户偏好。
*   **规划层**：将复杂目标拆解为可执行的子任务序列。
*   **行动层**：通过 Function Calling 调用外部 API 或执行代码。

## 5.1.2 ReAct 算法实战

ReAct（Reasoning + Acting）是目前最主流的 Agent 算法，其核心是“思考-行动-观察”的循环：

```
Thought: 我需要先获取上周的销售数据。
Action: query_sales_api(start_date="2026-04-09", end_date="2026-04-15")
Observation: {"total_sales": 50000, "growth": "+15%"}
Thought: 接下来需要将这些数据绘制成柱状图。
Action: generate_chart(data=..., type="bar")
...
```

## 5.1.3 任务分解策略

对于超长链路任务，建议采用 **Plan-and-Solve** 模式：先让大模型生成一份详细的执行计划，确认无误后再逐步执行。这能有效减少 Agent 在复杂逻辑中的“迷路”现象。

> **技术决策点**：在阿里云百炼平台上，可以通过配置“智能体应用”快速实现 ReAct 流程，无需从零编写底层调度逻辑。
