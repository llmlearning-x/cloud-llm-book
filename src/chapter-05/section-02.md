# 5.2 工具调用与记忆系统设计

## 5.2.1 Function Calling 机制

Qwen3.6-Plus 原生支持 Function Calling，能够精准地将自然语言指令转化为结构化的 API 调用参数。

**定义工具 Schema：**
```json
{
  "name": "get_weather",
  "description": "查询指定城市的天气",
  "parameters": {
    "type": "object",
    "properties": {
      "city": {"type": "string"},
      "date": {"type": "string"}
    },
    "required": ["city"]
  }
}
```

**执行流程：**
1.  **模型识别**：大模型根据用户意图决定是否需要调用工具。
2.  **参数提取**：模型生成符合 Schema 定义的 JSON 参数。
3.  **本地执行**：系统在本地执行对应的 Python 函数并获取结果。
4.  **结果回填**：将执行结果作为 `role: tool` 的消息返回给模型，由其生成最终回复。

## 5.2.2 记忆系统架构

Agent 的记忆能力决定了其交互的连贯性和个性化程度：

*   **短期记忆（Working Memory）**：通过滑动窗口管理当前对话的上下文，确保多轮对话的逻辑不中断。
*   **长期记忆（Long-term Memory）**：利用向量数据库存储用户的偏好、历史操作记录等关键信息。例如，当用户说“帮我订一张去北京的票”时，Agent 能通过长期记忆自动补全“出发地：南京”和“舱位：商务座”。

## 5.2.3 阿里云百炼 Agent 开发

在百炼平台中，开发者可以通过可视化界面为 Agent 绑定多种插件（如钉钉消息发送、高德地图查询、企业内部 CRM 系统等）。平台会自动处理复杂的鉴权和参数传递逻辑，让开发者专注于业务逻辑的实现。

> **最佳实践**：在设计工具时，务必提供清晰、无歧义的 `description` 和 `parameter` 说明，这是提升 Agent 调用成功率的关键。
